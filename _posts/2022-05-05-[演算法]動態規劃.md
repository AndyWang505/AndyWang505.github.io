## 動態規劃(Dynamic Programming)，簡稱DP

屬於分治法的延伸，主要可以將原問題分解為相較簡單的子問題，再通過子問題的解求出複雜問題的方法。

動態規劃的核心可以說是 **記憶化儲存**，每當子問題的答案被算出來時，同時將答案儲存在記憶體中，以便下次需要時可以直接透過查表的方式得出解，並從而減少計算量。

## 舉例 N!

> 5 * 4 * 3 * 2 * 1 = 120

### 暴力法
直接用for迴圈計算 N!

```c++
const int N = 10;

void factorial(){
    int f = 1;
    for (int i=2; i<N; ++i)
        f *= i;
}
```

### DP
透過動態規劃將所有f[N]儲存起來，有點像用空間換取時間的概念

```c++
const int N = 10;
int f[N];

void factorial(){
    f[0] = 0;
    f[1] = 1;
    for (int i=2; i<N; ++i)
        f[i] = f[i-1] * i;
}
```

## 費波那契數列(Fibonacci)

另外一個舉例是Fibonacci

![](https://i.imgur.com/gy3sYKG.png)

簡單的說就是 從第一次等於0，第二次等於1開始，下一個數字等於前兩個數字的相加，以此類推...

| -------- |
| f(0)=0 | 
| f(1)=1 | 
| f(2)=0+1=1| 
| f(3)=1+1=2 | 
| f(4)=1+2=3 | 
| f(5)=2+3=5 | 
| f(6)=3+5=8 | 
| ... |

所以結果得出就是 F(n) = F(n-1) + F(n-2)

那如果我們用程式遞迴來解的話

```c++
F(long n){
    if(n <= 2) return 1;
    return F(n-1) + F(n-2); 
}
```
所以可以發現時間複雜度是Ｏ(2^n)，所以這個效率是十分不佳的

若換用DP解的話

```c++
int main(){
    long long int F[50];
    int n;
    F[0]=0;
    F[1]=1;
    for (int i=2;i<50;i++){
        F[i]=F[i-1]+F[i-2];
    }
    while(cin>>n){
        cout<<F[n-1]<<endl;
    }
}
```
可以發現時間複雜度變Ｏ(N)，跟遞迴作法的效率實際上差很多，但也是有好有壞

遞迴在讀寫上會比好比較直覺一點，但換來的就是效率差

通常在實作也是可以先求有再求好啦XD，最後再慢慢優化就行了

## 複雜度

### 時間複雜度 BigＯ(N)

> 通常使用for迴圈來做，N個問題共會花費Ｏ(N)時間

### 空間複雜度

BigＯ(N)

> 需要空間來儲存每次計算得出的答案

BigＯ(1)

> 若使用暴力法就不需要空間來儲存了，直接計算得出結果...
