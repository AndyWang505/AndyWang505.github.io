[資料結構] 鏈結串列(Linked List)

By AndyWang505
March 20, 2022 , Posted 2022-03-20 Updated 2022-05-06 4 min read

前言

鏈結串列(Linked List)在資料結構中屬於常見的資料儲存方式

顧名思義就是把資料串在一起，並在資料中具備它們的索引值(index)

陣列與鏈結串列

鏈結串列與陣列很相似但用途完全不同

　陣列(Array)鏈結串列(Linked List)
記憶體配置連續不連續
記憶體空間使用彈性小彈性大
資料存取需移動整筆資料可任意插入或改變指向
資料型態皆相同每個節點可能不同
存取速度快慢
存取空間需求小大
記憶體浪費易不易

鏈結串列組成由 標頭(header) 和 節點(node)，像這個鏈結指向下一個節點，最後一個節點會指向空值

單向、雙向、迴圈鏈結串列

鏈結串列又分成單向鏈結串列和雙向鏈結串列

  • 單向鏈結串列：節點若無指向上個節點相連，只能做單向遍歷，稱作單向鏈結

  • 雙向鏈結串列：節點有指向上個節點並相連，能做雙向遍歷，稱作雙向鏈結

  • 迴圈鏈結串列：首節點與末節點相連，稱作迴圈鏈結

指標概念

在鏈結串列中其實就是使用指標的方式來動態的配置記憶體位址

所以在繼續往下前需要先提到指標(pointer)的概念，就是c語言特有的指標特性

以生活化方式解釋就是，如果今天要交付一個報告，而我提供了一個網址，這網址連結導向到報告，而網址指向的就是這個資料的「指標」，也就是資料的位址

EX. 以程式舉例

以程式來舉例的話

void main(){
    int a = 5;
    int b = 10;
    int c = 28;

上面宣告了3個變數，int a = 5為例，在程式開始執行後，程式便會去記憶體中要一塊儲存空間，並把5這資料放進去，而5會有對應的記憶體位址

回到剛剛生活化的解釋就是，網址就是變數位址，報告就是變數值，報告的標題就是變數名稱

鏈結串列的插入與刪除

插入

假設有一個鏈結串列MOU，今天有一個節點C需要新增，則過程只需要將O到U的指向改成O指向到C，再由C指向到U就好了，完全不需要移動到資料

刪除

假設有一個鏈結串列MOCU，今天需要刪除C這個節點，會先找到C節點，並在將前一個節點的指向到C節點後的節點，有點像跳過它的感覺
